FingerPrints

When using the internet, users expect a secure connection to whatever service they are using. These secured connections were not present in the early days of the internet, but for modern day internet users, the HTTPS web connection is common. When introduced, HTTPS had two primary purposes, one being the secure, authenticated, and encrypted connection between the user and the web server. The other purpose was to prevent eavesdropping by third parties, however private institutions have gotten around this aspect of HTTPS connections.


HTTPS proxy appliances are devices that allow private institutions to circumvent internet privacy and security. They do this by adding a pseudo certificate authority to their users' browsers. This circumvents the certificate authority system and allows them to monitor every internet action on their sites. All it takes is for a company to install a SSL proxy to add the certificate. Eavesdropping on private connections is also known as a MITM attack, or man in the middle attack. MITMs is when a third party secretly relays communications between two parties that believe they are directly communicating with each other. These attacks are usually prevented by certificate authentication. MITM attacks do pose a risk however. Some companies have been seen using MITM attacks, such as NOKIA. In 2013 NOKIA was found using proxy servers to decrypt HTTPS traffic. The company had to release a statement stating the content was not stored permanently.


Unfortunately, these SSL interceptions cannot be entirely prevented, but they can be reliably detected, as completely spoofing a certificate is impossible. This is thanks to public and private keys.SSL security provides the public cryptographic key but not the private key, which is not shown to the public. The certificate's public key is used to encrypt data that can only be decrypted by its matching private key. SSL proxies are forced to use different public keys, as they do not have the matching private keys.Certificate Authorities are trusted authorities that issue SSL certificates.The same certificates used to link entities with public keys.The CA is responsible for verifying the trustworthiness of a website, as well as its authentication.


Those inspecting SSL certificates can create a cryptographic hash of the certificate. A hash is a complex algorithm that detects changes in a certificate, or “fingerprint” them. If even the smallest change is detected, the algorithm completely changes it into another completely different fingerprint. SHA1 or secure hashing algorithm 1 is used to today to detect changes in security certificates


There are things to consider when testing a certificate's fingerprint. Larger companies may use multiple certificates on their websites. This leads to people getting different fingerprints and assuming they are being intercepted. In some cases, it is required for communications to be intercepted or monitored. Federally funded schools are required to have a plan for monitoring students' lives. This is to make schools safer at the cost of privacy.
